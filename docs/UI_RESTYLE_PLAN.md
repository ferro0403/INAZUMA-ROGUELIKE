# UI Restyle Plan

## Obiettivo del restyle
- Definire e realizzare progressivamente il restyle dell'interfaccia utente mantenendo invariati gameplay e logiche esistenti.

## Principi non negoziabili
- Nessuna modifica al gameplay durante il restyle.
- Nessuna modifica a dataset, probabilità, seed e salvataggi.
- Mobile e desktop devono essere progettati separatamente.
- Le Player Card devono avere varianti centralizzate e coerenti.
- Evitare CSS globale distruttivo.
- Evitare rerender generali per interazioni locali.
- Preservare accessibilità, leggibilità e aree touch.
- Non ricostruire l'applicazione da zero.
- Non migrare a React o ad altri framework durante il restyle.
- Il repository corrente resta la fonte tecnica finale.
